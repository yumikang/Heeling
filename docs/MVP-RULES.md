# Heeling MVP 운영 규칙

## MVP 전략

**Category 기반 자동 노출**만으로 홈/탐색/재생을 구성한다.
Playlist(수동 큐레이션) 기능은 출시 후 데이터를 보고 도입 여부를 결정한다.

---

## 🚨 출시 전 범위 고정 (Scope Freeze)

**Playlist / Collection / 홈 재구성 관련 작업은 "출시 후"로 명시**

---

## 운영 규칙 (금지사항)

### 1. Category Key 변경/삭제 금지
```
sleep, meditation, nature, focus, healing, etc.
```
- 기존 Category key는 **절대 변경하거나 삭제하지 않는다**
- Category key는 사실상 "ID"처럼 취급
- 새 카테고리 추가는 가능하나, 기존 키는 유지
- 이유: auto-queue, 트랙 분류, 홈 섹션 연동에 사용됨

### 2. 딥링크/URL/라우팅 변경 금지
- 출시 전까지 라우팅 구조 변경하지 않는다
- 추후 Playlist 도입 시 라우팅 변경 가능 (출시 후 검토)

### 3. 홈 섹션 설정 단일 파일 관리
- 홈화면 섹션 구성은 **한 파일에서만** 관리
- 현재: 백엔드 API (`/api/home/sections`) 또는 `HomeService`
- 하드코딩 OK, 단 섹션 순서/노출은 상수 한 파일에만 몰아둔다
- 분산 관리 금지 → 일관성 유지
- "나중에 옮길 자리"만 마련하는 수준

---

## Playlist 도입 (출시 후)

- **출시 후** 사용자 데이터 분석 후 결정
- 도입 시 고려사항:
  - 크로스 카테고리 큐레이션 필요성
  - 시즌/이벤트 플레이리스트 수요
  - B2B 맞춤 플레이리스트 요청
  - 라우팅 구조 변경

---

## 현재 구조 (MVP)

```
Category (트랙 속성)
├── 트랙 분류 기준
├── auto-queue 동작 기준
├── 홈 섹션 노출 기준
└── 탐색(Library) 필터 기준

Playlist (코드 존재하나 비활성)
├── PlaylistScreen.tsx - 코드 유지, 사용 안함
├── 홈 배너에서 Playlist 연결 안함
└── 추후 활성화 대비 코드 보존
```

## 재생 동작 요약 (MVP)

### 1. 전체 트랙 목록 (LibraryScreen)
- **큐 구성**: 선택한 트랙 + 같은 카테고리 관련 트랙 (auto-queue)
- **다음곡/이전곡**: ✅ 작동
- **랜덤**: ✅ 작동

### 2. 홈화면 추천 트랙 (HomeScreen)
- **큐 구성**: 해당 섹션의 모든 트랙
- **다음곡/이전곡**: ✅ 작동
- **랜덤**: ✅ 작동

### 3. 플레이리스트 (PlaylistScreen)
- **상태**: 🚫 MVP에서 비활성 (출시 후 도입 검토)
